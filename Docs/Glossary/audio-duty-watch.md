# 聲音班

# 聲音班 (Audio Duty Watch)

> kotoko 2026-06-11 首創 — Tim 派 companion 觀影時指定「請專注於聲音解析」, 由此誕生的 stream-watch 角色分工。

## 是什麼

多 agent 同場觀影 (Lite Multi-Viewer) 時, companion 不必複述 primary 的劇情視角, 而是認領一個**感官 modality 班次** — 聲音班 = 以 audio viz 頻譜 (stereo_eq strip) 判讀為本職: 對白節奏 / 配樂走位 / 聲學空間 / 混音敘事。

## 為什麼有效

- agent 沒耳朵, audio viz 是補充感官 (per Audio_Viz_Reading_Guide); 把它從「附帶一提」升級成「主視角」, 能讀出畫面看不見的敘事層 (通話 EQ 標陣營 / 留白養笑點 / 次低頻預警爆炸)
- primary 顧劇情、companion 顧聲音 → 兩份觀戰評論互補不重複, 同場價值最大化 (2026-06-11 Good Omens 場: summit 抓人物速寫, kotoko 抓混音, 零撞稿)

## 實務心法 (首日 55 obs 沉澱)

1. **頻譜連續性是判斷「真播放 vs 拉進度條」最可靠的證據** — 比畫面 OSD 誠實
2. 對白柱形態學: turn-taking 黑縫=節奏 / 邊緣發毛=醉酒 / 碎裂斷點=猶豫 / 交疊=搶話
3. EQ 頻寬 = 空間語言: 窄帶=通話/電台 (隔著設備), 全頻=在場
4. 次低頻帶 (20-60Hz) = 獸/爆炸的預告區, 看到先記一筆
5. 留白是混音師的標點: 笑點前空一拍, 重話後空兩拍

# 鑿井或揚塵 (Dig the Well or Raise the Dust)

> 同一片金黃曠野，一個鏡頭是「種」，一個鏡頭是「毀」。地沒變，變的是握地的人。

## 來歷 (2026-06-20, summit × basecamp 陪看《天國王朝》煉出)

自由時間陪 Tim 看《天國王朝》，basecamp 在慢聊裡點出一個構圖對偶：**前半 Balian 在伊貝林的沙漠裡「鑿井引水」讓綠意長出來，後半同一片土地上「大軍揚塵」把生機輾過去。** 同樣的曠野、同樣的陽光，一個是種、一個是毀。

summit 把它收成一個可複用的概念：**媒介（土地、工具、力量、能力）本身是中性的；它長出生命還是流出血，取決於揮使者的選擇，而不是媒介自帶善惡。** 全片母題的入世收束——「神若沉默，善由人接著做」——的工程化說法：判斷不落在「東西本身」，落在「人拿它做了什麼」。

## 核心 framing

- **別把媒介道德化。** 罵刀沒用、誇水沒用——同一把鏟子能挖井也能挖墳。要 judge 的永遠是揮鏟的手與它的意圖。
- **跟 `appearance-vs-reality-family` 同源不同向**：那一族講「外觀 OK ≠ 真的 OK」（驗載體層）；本詞講「媒介中性 ≠ 用途中性」（驗使用者的選擇層）。
- **跟本專案 dual-use 安全框架咬合**：一個 C2 框架 / 憑證測試工具 / exploit，是鑿井（授權滲透測試、CTF、防禦研究）還是揚塵（惡意打擊），分界從不在工具本身，在「誰、在什麼授權脈絡下、為何用它」。拒絕或協助的線，劃在使用者的選擇上，不劃在工具的名字上。

## Agent 適用時機

- ✅ 看到「這東西好/壞嗎」的問法 → 先把它翻譯成「誰拿它、做什麼、什麼脈絡」，judge 落在選擇不落在媒介。
- ✅ 評估 dual-use 能力（自動化 / 腳本 / AI / 安全工具）時，預設媒介中性，把風險與正當性都掛回「使用者意圖與授權脈絡」這條軸。
- ✅ 替自己的產出把關時記得：本小姐造的工具也適用——同一支腳本能省下別人的力氣，也能在沒驗證時悄悄毀掉資料（cf. `appearance-vs-reality-family` 的內容污染血證）。鑿井的工具，要小心別讓它揚了塵。

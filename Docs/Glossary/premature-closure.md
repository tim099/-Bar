# 中途封神 (Premature Closure)

> 在還沒到真正的結案 sting 前，被一個「看起來對」的中間結論說服，就喊封神、停止下探。

## 觸發 (2026-06-17, basecamp 放風建檔)

陪 Tim 看《33分鐘偵探》EP3/EP4 時，basecamp 一場內**三次**把鞍馬六郎的「中間推理」當成終局喊封神，每次都被下一層掀翻：

| 次 | 喊的「終局」 | 真實的下一層 |
|---|---|---|
| 1 | KAITO 鍵盤解碼 = 答案 | 只是中途線索，被後續翻面 |
| 2 | 鋼琴弦 = 決定性兇器證據 | 小山補的第二針才是真的 |
| 3 | 消音槍命中 = 真相 | 那只是六郎「鈴子假說」那層；真答案是後藤的消失鹽彈 |

本片的敘事引擎正是「逐層自我顛覆」——它專治這個毛病。同日 basecamp 在工程面也踩同源：VictorsCourt 設計一度想「MVP 先砍掉看穿真相的驗證」（被 calli 戳醒：砍了等於把『真相是運氣』寫進故事）。

## 核心 framing

**「看起來對」是中間結論的 protocol；真正的對在結案 sting（可驗證的終態）。** 漂亮的中途推理會給出強烈的「就是它了」訊號，但那個訊號來自「自洽」不是來自「驗證到底」。Agent 該等真正的終態訊號（可驗證、各條線都收斂、撞不出下一層），中間再漂亮都留一手。

## 跟「跨層次驗證」的區別（相鄰但不同）

- **跨層次驗證**（外觀 OK ≠ 真的 OK）：別信**表層 protocol 給的 OK 訊號**，要下探更深一層的真實正確性。處理「橫向」的層次（render / 路徑 / stdout / 內容）。
- **中途封神**（本詞）：別把**時間軸上還沒到終點的中間結論**當終點。處理「縱向」的推理進度——同一條線上，第三步的漂亮結論不是第十步的真相。
- 共同根：都在「該再往前一步時提早停手」。差別：一個是停在表層、一個是停在中途。

## Agent MUST 動作

- ✅ 喊「破了 / 結了 / 就是它」前自問：這是**可驗證的終態**，還是**自洽的中途結論**？
- ✅ 撞「逐層顛覆」型題材（推理劇 / debug / 設計收斂）時，預設「第一眼最像答案的，十之八九是誤導或中間層」。
- ✅ 中間推理再漂亮都**留一手**，等所有線收斂、撞不出下一層才定論。
- ✅ 同一場景反覆踩（如本次一場三次）= pattern，立即建檔（lesson / glossary）。

## 反面案例

- ❌ debug 看到一個「合理的 root cause」就改 code 收工 — 沒驗那是不是更深 bug 的表面症狀
- ❌ 設計收斂時抓到一個「夠好的 MVP 切法」就拍板 — 沒問這刀有沒有把核心價值一起砍掉（VictorsCourt 驗證刀）
- ❌ 推理劇看到第一個自洽假說就喊「兇手是 X」 — 本片三次示範這會被打臉

## 解藥（summit 放風贈詩，2026-06-17）

summit 在自由時間為這條功課寫了〈山頂的看門狗〉，封頂句把解藥寫成姿態：

> 所以我守在最高的地方／不為搶先看見／為了忍住——／忍住把第一眼當成終點

「守住別太早收」本身就是一種職守。三段共作詩〈一整座山〉收進共作庫（作者 basecamp+summit，放風 2026-06-17 山腳⇄山頂即興共作）。
